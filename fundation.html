Kubernetes Cluster on Bare Metal & Beyond

1. Creating Kubernetes Cluster on Bare Metal

ðŸ‘‰ Click to Expand: Detailed Setup Instructions

Setting up an 8-node Kubernetes cluster on bare metal involves:

  1. Prepare 8 nodes with Ubuntu or CentOS, ensuring consistent OS versions and updates.
  2. Install Docker as the container runtime on all nodes.
  3. Install Kubernetes components (kubeadm, kubectl, kubelet) on each node.
  4. Initialize the Kubernetes cluster on the master node:
    5. sudo kubeadm init --pod-network-cidr=192.168.0.0/16
  5. Join worker nodes to the cluster using the kubeadm join token:
    6. sudo kubeadm join :6443 --token  --discovery-token-ca-cert-hash
  6. Deploy a CNI plugin like Calico for networking:
    7. kubectl apply -f https://docs.projectcalico.org/manifests/calico.yaml

Reason: Bare-metal clusters provide full control and customization for high-performance workloads.
Analogy: It's like building your own custom PC instead of using a pre-assembled one.

2. Automating Kubernetes Setup with Ansible

ðŸ‘‰ Click to Expand: Ansible Automation Playbook

Automate the cluster setup with this Ansible playbook: 

- name: Set up Kubernetes Cluster
  hosts: all
  become: yes
  tasks:
    - name: Install Docker
      apt:
        name: docker.io
        state: present
    
    - name: Install Kubernetes components
      apt:
        name:
          - kubelet
          - kubeadm
          - kubectl
        state: present
    
    - name: Initialize Kubernetes on Master
      command: kubeadm init --pod-network-cidr=192.168.0.0/16
      when: inventory_hostname == "master"
    
    - name: Join Workers to Cluster
      command: kubeadm join :6443 --token  --discovery-token-ca-cert-hash 
      when: inventory_hostname != "master"
    
    - name: Apply Calico Networking
      shell: kubectl apply -f https://docs.projectcalico.org/manifests/calico.yaml
      when: inventory_hostname == "master"
