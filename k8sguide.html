Setting Up Kubernetes on Bare Metal Nodes

A detailed guide to deploying a high-availability Kubernetes cluster with containerd and HAProxy.

Cluster Configuration and Components

We will set up an 8-node Kubernetes cluster with the following components:

  • Master Nodes: master1, master2, master3
  • Worker Nodes: worker1, worker2, worker3
  • Load Balancers: loadbalancer1, loadbalancer2

Component Versions

  • Kubernetes: kubeadm, kubelet, kubectl v1.27.0
  • Container Runtime: containerd v1.6.12
  • Load Balancer: HAProxy v2.8
  • Pod Networking: Calico (latest)
Step 1: Load Balancer Setup
  1. Install HAProxy on both load balancers: 
    sudo apt install -y haproxy
  2. Edit the HAProxy configuration: 
    
frontend kubernetes_frontend
    bind *:6443
    default_backend kubernetes_backend

backend kubernetes_backend
    balance roundrobin
    server master1 <Master1-IP>:6443 check
    server master2 <Master2-IP>:6443 check
    server master3 <Master3-IP>:6443 check
  3. Restart HAProxy: 
    sudo systemctl restart haproxy
Step 2: Node Pre-Configuration

    sudo apt update && sudo apt upgrade -y

Step 3: Install containerd
  1. Install containerd: 
    sudo apt install -y containerd
  2. Configure containerd: 
    
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml
sudo systemctl restart containerd
Step 4: Install Kubernetes Components
  1. Add Kubernetes repository: 
    
curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
echo "deb https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list
  2. Install Kubernetes packages: 
    sudo apt update && sudo apt install -y kubelet kubeadm kubectl
Step 5: Cluster Initialization and Joining

Initialize the First Master Node


sudo kubeadm init --control-plane-endpoint "<LoadBalancer-IP>:6443" --upload-certs --pod-network-cidr=192.168.0.0/16

Join Other Masters

sudo kubeadm join <LoadBalancer-IP>:6443 --control-plane ...

Join Worker Nodes

sudo kubeadm join <LoadBalancer-IP>:6443 --token ...
Step 6: Verify the Cluster 
kubectl get nodes
kubectl get pods --all-namespaces
